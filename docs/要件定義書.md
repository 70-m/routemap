# RouteMap 要件定義書

**バージョン:** 1.0  
**作成日:** 2026-04-02  
**対象ファイル:** index.html

---

## 1. アプリ概要・背景

RouteMap は、スマートフォン・PC のブラウザから使用できる Web ナビゲーションアプリケーションである。  
Claude Code を用いた個人学習目的で開発されており、一般ユーザーが利用できる品質を目指す。

外部サーバーへの依存は最小限とし、クライアントサイドのみで動作する設計とする。

---

## 2. 対象ユーザー・デバイス

| 項目 | 内容 |
|------|------|
| ユーザー | 個人利用（開発者本人）、ただし一般ユーザーレベルの品質を維持 |
| 対象デバイス | Android スマートフォン / iOS スマートフォン / PC |
| 動作環境 | ブラウザ（Chrome / Safari / Firefox 等） |
| ネットワーク | 基本オンライン必須（地図タイル・ルート検索・ジオコーディング） |

---

## 3. 機能要件

### 3.1 ルート検索

| ID | 要件 |
|----|------|
| F-01 | テキスト入力による出発地・目的地の検索（Nominatim ジオコーディング） |
| F-02 | 地図タップによる出発地・目的地の設定 |
| F-03 | 移動手段の選択（自動車 / 自転車 / 徒歩） |
| F-04 | OSRM API によるルート取得（距離・所要時間・ターン案内付き） |
| F-05 | ルート表示（地図上にポリライン描画） |
| F-06 | ルートサマリー表示（距離・所要時間） |
| F-07 | ステップ一覧の表示（番号・矢印・道路名・距離） |
| F-08 | 出発地・目的地の入れ替え |
| F-09 | ルート・マーカーのリセット |

### 3.2 音声案内ナビゲーション

| ID | 要件 |
|----|------|
| F-10 | GPS による現在地取得（高精度モード） |
| F-11 | 3フェーズ音声案内（予告200〜400m前 / 直前50〜200m前 / 実行50m以内） |
| F-12 | Web Speech API による日本語 TTS（音声案内） |
| F-13 | 16種類以上のマニューバー対応（右折・左折・ロータリー・ランプ等） |
| F-14 | ルート逸脱検出（ルート座標から50m超で警告表示） |
| F-15 | 自動再ルート（逸脱検出時、12秒クールダウン付き） |
| F-16 | ミュート・アンミュート機能 |
| F-17 | 案内バナーの地図上オーバーレイ表示 |
| F-18 | 現在ステップのハイライト（一覧・地図マーカー） |
| F-19 | Screen Wake Lock による画面常時点灯 |
| F-20 | 到着検出と案内自動終了 |

### 3.3 デモ走行

| ID | 要件 |
|----|------|
| F-21 | ルート上を自動走行するデモモード |
| F-22 | 走行速度の調整（0.5×〜16×、2倍ステップ） |
| F-23 | 再生・一時停止・停止の制御 |
| F-24 | 進捗バーによる走行位置の表示 |
| F-25 | デモ中の音声案内（通常案内と同一ロジック） |
| F-26 | 走行軌跡の表示（黄色ポリライン） |

### 3.4 GPS 追跡（通常）

| ID | 要件 |
|----|------|
| F-27 | バックグラウンドでの GPS 継続追跡 |
| F-28 | GPS 精度円の地図表示 |
| F-29 | 現在地への地図センタリング |
| F-30 | GPS を出発地として自動設定するモード |

### 3.5 地図操作

| ID | 要件 |
|----|------|
| F-31 | OpenStreetMap タイルによる地図表示 |
| F-32 | ズームイン・ズームアウトボタン |
| F-33 | 地図タップによるマーカー配置 |
| F-34 | 案内中の地図自動追従（GPS / デモ位置） |

### 3.6 ログ・デバッグ機能

| ID | 要件 |
|----|------|
| F-35 | API リクエストのログ記録 |
| F-36 | OSRM レスポンスの構造化ログ（ウェイポイント・ステップ一覧） |
| F-37 | Raw JSON ログの表示 |
| F-38 | 音声発話ログの記録 |
| F-39 | エラーログの記録 |
| F-40 | ログエントリーの折りたたみ表示 |
| F-41 | ログのクリア機能 |

## 4. 非機能要件

| ID | 分類 | 要件 |
|----|------|------|
| NF-01 | アーキテクチャ | `index.html` 1ファイルのみで構成する。ファイル分割は禁止 |
| NF-02 | ライセンス | 著作権に抵触する実装・コードを一切使用しない |
| NF-03 | コスト | 有料ライブラリ・API を使用しない |
| NF-04 | レスポンシブ | スマートフォン（<900px）とデスクトップ（≥900px）の両レイアウトに対応 |
| NF-05 | コーディング規約 | コメントは日本語、ブロック単位で記述 |
| NF-06 | コーディング規約 | インデントはタブ |
| NF-07 | ログ | 処理の要所に `console.log` を入れる |
| NF-08 | ログ | エラー・失敗ケースには必ず `console.error` / `console.warn` を入れる |
| NF-09 | 開発フロー | Issue → ブランチ → 実装 → テスト（全項目OK） → PR → マージ確認 |
| NF-10 | ライブラリ管理 | 外部ライブラリのバージョンアップは勝手に行わない |

## 6. 制約事項

- ネットワーク接続が必要（地図タイル・ルート検索・ジオコーディング）
- GPS はユーザーの許可が必要
- Web Speech API は一部ブラウザ・OSで未対応の場合がある
- Screen Wake Lock はブラウザの対応状況に依存
- OSRM パブリックサーバーはリクエスト頻度制限があるため、連続アクセスを避ける
- Nominatim 利用ポリシーに従い User-Agent を付与して呼び出す
