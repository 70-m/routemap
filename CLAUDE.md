# RouteMap - CLAUDE.md

## プロジェクト概要

スマートフォン・PCのブラウザから使用するナビ系Webアプリケーション。
個人学習目的で開発しているが、一般ユーザーが使えるレベルの品質を目指す。

**対象デバイス:** Android / iOS / PC（ブラウザ）

---

## ファイル構成

- `index.html` 1ファイルのみ。ファイルの分割は絶対に行わないこと。

---

## コーディング規約

### コメント
- コメントは必ず**日本語**で記述する
- 処理のブロック単位でコメントを入れる（何をしているかを説明する）

### インデント
- **タブ**を使用する（スペース不可）

### ログ
- 処理の要所に適度な実行ログ（`console.log`）を入れる
- エラー・失敗ケースには**必ずログを入れる**（`console.error` / `console.warn`）

### GitHub への記載
- Issue・コミットメッセージ・PR・コメント等、GitHub へ記載するものは**すべて日本語**を使用すること

---

## ライブラリ・著作権

- 著作権に抵触する実装は絶対に行わない
- 有料ライブラリは使用しない
- 新規ライブラリの追加が必要な場合は、**必ずユーザーに確認を取ること**
- 既存ライブラリのバージョンアップは勝手に行わない
  - バージョンアップが必要な場合は**必ずユーザーに確認を取ること**
  - バージョンアップ時はアプリケーション全体のコードを確認し、関連する箇所を全て修正すること

---

## 開発フロー

アプリケーションを更新する際は、以下のフローを必ず守ること：

1. **Issue作成** - GitHub上でIssueを作成する
2. **ブランチ作成** - Issueに対応するブランチを作成する
3. **実装** - ブランチ上でファイルを更新する
4. **バージョン更新** - `APP_VERSION` を `YYMMDD + PRxx`（例: `260420PR47`）形式で更新する
5. **テスト** - 後述のテストを実施し、全項目がOKになったことを確認する
6. **プルリクエスト** - 全テストOK後にPRを作成する
7. **マージ確認** - マージは必ずユーザーに確認を依頼し、自動でマージしない

---

## テスト

アプリケーション更新時は、以下の全テストを実施すること。
テスト項目は今後追加されていく。

### 自動テスト（マージ前・Claude が実行）

実装後、PR 作成前に Claude が自動で確認する。全項目 OK になってから PR を作成すること。

- JS 構文エラーがないか
- 必須 DOM 要素がすべて存在するか
- 必須関数がすべて実装されているか
- タブインデント規約に準拠しているか（スペースインデント0件）
- `console.log` / `console.error` / `console.warn` が実装されているか
- `catch` ブロック全てに `console.error` / `console.warn` が設定されているか
- 外部ライブラリのバージョンが固定されているか
- 有料ライブラリが使用されていないか
- Nominatim の `User-Agent` が設定されているか

### 手動テスト（マージ後・ユーザーが実行）

マージ・公開後にブラウザで動作確認する。

- ページが正常に読み込まれるか
- GPS取得が動作するか（位置情報の許可）
- ルート案内が正常に開始・終了できるか
- オフライン時に適切なエラーが表示されるか
- デモ走行で自車が正常にルートの最後まで移動するか
